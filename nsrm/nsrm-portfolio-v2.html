N-SRM은 삼성전자의 기존 글로벌 구매관리 시스템(G-SRM)을 대체하기 위해 개발된 차세대 통합 구매 시스템입니다. DX(디바이스)와 DS(반도체) 부문으로 분리 운영되며, MFA 인증과 대용량 데이터 처리를 강화했습니다.

본 케이스는 사내 바이어(구매 담당자) 시점에 집중합니다. 협력사 측은 별도 포털로 분리 운영되며, 한 페르소나의 동선을 끝까지 따라가는 의사결정의 일관성에 초점을 두었습니다.

Design Decisions

시니어의 판단이 드러나는 4가지 결정

11개 의사결정 중 디자인 사고의 깊이를 가장 잘 보여주는 네 가지를 선별했습니다.

#2
Strategic

사용자 페르소나: 협력사 vs 바이어 — 바이어로 통일

N-SRM은 협력사(외부)와 바이어(사내) 두 페르소나를 지원합니다. 포트폴리오에서 어느 시점을 다룰지가 첫 번째 전략적 결정이었습니다.

A. 두 페르소나 병존 B. 바이어 시점 통일 ✓
핵심 근거: 시니어 채용 시그널의 핵심은 "한 페르소나의 동선을 끝까지 따라가는 의사결정의 일관성". GNB 메뉴 구성이 이미 글로벌 바이어 시스템 패턴과 일치 — 5개 카피 수정만으로 전환 완료.

트레이드오프: 협력사 포털 역량 시그널 손실 → 케이스 인트로에 한 줄로 보완

#7
UI

Hero Banner: viewport 30%의 빌딩 이미지를 제거

원본 메인의 히어로는 viewport 30%를 차지하는 빌딩/숲 사진 + "Purchasing The Future" 카피였습니다.

A. 유지 (브랜드 자긍심) B. 제거 + 운영 콘텐츠 ✓
핵심 근거: B2B 운영 도구의 첫 화면은 "오늘 무엇을 해야 하는지"가 우선. Linear, Notion, Stripe Dashboard 같은 현대 B2B SaaS — 첫 viewport에 actionable data. 제거한 자리에 4 우선순위 카드 도입.

트레이드오프: 브랜드 자긍심 시그널 손실 → 운영 도구의 본질은 효율

#8
UI

Action Cards: 정보량 0 → 정보 밀도 6배

원본 메인의 액션 카드 5개는 아이콘 + 제목 + 화살표만 표시하는 순수 메뉴 링크 형태였습니다.

A. 메뉴형 유지 B. 데이터형 변환 ✓
핵심 근거: 원본 정보량 0개(제목만) → v7 카드: "4건 / 평가 2·승인 1·검수 1 / 17:00 마감 / D-DAY" — 동일 면적에서 정보 밀도 6배.

트레이드오프: 시각 복잡도 증가 → 파워유저 대상이므로 허용 범위

#5
System

필터 위계: 13개 평면 나열 → 3단 분리

협력사 리스트에 필요한 필터가 13개. 성격이 다른 필터들이 같은 행에 나열되어 있었습니다.

A. 평면 나열 B. 3단 위계 분리 ✓
핵심 근거: 스코프(사용자 컨텍스트) / 부문(비즈니스 컨텍스트) / 일반(데이터 조건) — 성격이 다른 필터는 시각적 무게도 달라야 함. Toggle / Segment / Dropdown으로 차등 배치.

트레이드오프: 수직 공간 소모 → 인지 비용 절감이 훨씬 큼
